**Your resume is a filtering document, not a biography.** Hiring managers spend less than 30 seconds on a first pass. Applicant Tracking Systems (ATS) filter resumes based on keyword matches before a human ever sees them. An IT resume that works is structured clearly, keyword-rich, and quantified. One that does not work is a walls-of-text job description copy.

*For a comprehensive deep-dive into resume optimization, ATS strategy, and tailoring, see the [Resume Optimization Course](/resume-optimization-course-start/).*

## IT Resume Structure

Use a clean, single-column format with these sections in order:

1. **Name and contact information** — name, phone, email, LinkedIn URL, GitHub/portfolio URL. No photo, no address beyond city and state.
2. **Professional summary** — 2–3 sentences. State your specialization, top 2–3 technologies, and what you bring. Write this last.
3. **Certifications** — list active certifications with exam code and year earned. Early-career professionals should put this section above work experience.
4. **Technical skills** — a structured list of platforms, tools, and technologies. Group by category (Operating Systems, Cloud, Networking, Security, etc.).
5. **Work experience** — reverse chronological. Each role gets a title, company, location, dates, and 3–6 bullet points.
6. **Projects / Home Lab** — crucial for early-career candidates. Describe what you built and what it demonstrated.
7. **Education** — degree, institution, graduation year. Less important than certs for most IT roles, but include it.

## Writing Experience Bullets

**Bad:** "Responsible for server maintenance and user support."

**Good:** "Managed 47-server Windows Server 2022 environment, achieving 99.8% uptime across 18 months through structured patch management and proactive monitoring."

Every bullet should follow the pattern: **[Action verb] + [what you did] + [quantified result or scope]**.

Strong IT action verbs: *Administered, Deployed, Configured, Automated, Migrated, Reduced, Implemented, Monitored, Secured, Troubleshot, Provisioned, Designed.*

## ATS Keywords

Applicant Tracking Systems filter on exact keyword matches. For each application:

1. Copy the job description into a text document.
2. Identify the top 10 technical terms (specific tools, platforms, protocols, certifications).
3. Ensure each appears verbatim in your resume — if you have the skill, use their exact terminology.

Do not stuff keywords without experience to back them. ATS gets you the interview; the interview tests whether the resume was honest.

## Certifications Section Best Practices

| Do | Do Not |
|----|--------|
| List active certifications with exam code | List expired certifications unless actively renewing |
| Include year earned | Include certifications in progress without noting "In progress" |
| Put near the top if early-career | Bury certifications at the bottom for associates and seniors |

## Home Lab / Projects Section

This section is what separates candidates with no professional IT experience from those with demonstrated skills.

**Example entries:**

- *Three-node Proxmox VE cluster with shared Ceph storage. Implemented live VM migration, high availability fencing, and automated backup to a Proxmox Backup Server instance.*
- *Self-hosted Active Directory domain with Windows Server 2022. Configured Group Policy, DNS, DHCP, and certificate authority. Documented all configurations in a private wiki.*
- *Deployed Prometheus + Grafana monitoring stack on Rocky Linux 9, monitoring 8 VMs with custom alert rules and email notifications.*

Each entry should state what you built and what it demonstrates about your skills.

## Tailoring for Each Application

A generic resume performs poorly. An IT resume tailored to the job description performs significantly better.

The minimum tailoring per application:
- Rewrite the professional summary to mirror the role's title and top requirements
- Adjust the order of your technical skills to put the most relevant listed first
- Verify the ATS keywords from the job description appear in your resume
